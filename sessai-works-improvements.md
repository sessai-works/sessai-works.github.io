# SessAI Works — トップページ改善指示書

## 対象ファイル

`index.html`（https://sessai-works.github.io/ のトップページ）

---

## サイト概要・前提

- GitHub Pages でホストされた静的サイト
- ポートフォリオサイトとしての構造（ナビ・複数ページ）は維持する
- トップページは LP（ランディングページ）的な役割も担う
- 既存のデザイントーン・レイアウトを崩さないこと

---

## 改善点一覧（全5点）

---

### 改善①：ファーストビューに CTA ボタンを追加

**場所：** ヒーローセクション（キャッチコピー・数字の下）

**内容：**
既存デザインのトーン・スタイルに合わせた CTA ボタンを1つ追加する。

- ボタンテキスト例：`まずは相談する →`
- リンク先：お問い合わせフォーム（`https://forms.gle/avxAvwP63AWKmj4L9`）
- デザイン要件：
  - 既存サイトで使用されているボタンスタイル・カラー・フォントに統一する
  - ヒーローセクション内で浮かず自然に馴染む配置にする
  - スマートフォン表示でも十分なタップ領域を確保する

---

### 改善②：Results セクションの直後に CTA を追加

**場所：** Results セクション（実績紹介）の終わり〜Process セクションの始まりの間

**内容：**
実績を読んで気持ちが動いた直後に次のアクションを促す。

- ひと言テキスト例：`こんな仕組み、あなたの会社にも作れます。`
- ボタンテキスト例：`まずは相談する →`
- リンク先：`https://forms.gle/avxAvwP63AWKmj4L9`
- デザイン要件：
  - 改善①のボタンと同じスタイルで統一する
  - セクション間に自然に溶け込む余白・配置にする

---

### 改善③：ターゲット一文を課題・状態ベースに変更

**場所：** ヒーローセクションのキャッチコピー直下（現在「バックオフィス業務を抱える中小企業・スタートアップの方へ」となっている一文）

**内容：**
属性（業種・規模）による絞り込みではなく、課題・状態への共感を呼ぶ表現に変更する。これにより、バックオフィスだけでなく営業・マーケ・経営など複数部門の担当者にも刺さるようにする。

- 変更後テキスト案：
  > 同じ作業の繰り返し、情報の散らばり、対応の抜け漏れ——部門を問わず、そのムダをAIで仕組みごと解決します。

- デザインは既存のスタイル（フォントサイズ・色）を維持する

---

### 改善④：Problem セクションに他部門の課題を追加

**場所：** Problem セクション（「こんな業務に、時間を取られていませんか？」のあるある一覧）

**内容：**
現状はバックオフィス寄りの課題が中心のため、営業・マーケ寄りの課題を1〜2個追加して読者層を広げる。

**現状の4項目（変更なし）：**
1. 毎月同じレポートを手作業で作成している
2. チャットやメールの対応が追いつかない
3. マニュアルや議事録を作るのに何時間もかかる
4. 社内ルールやナレッジを探すのに時間がかかる

**追加する項目（案）：**

```
📊
商談・顧客データを活かしきれていない
個別のメモや報告書が散在し、傾向把握・次の一手が後手に回る
```

```
📣
施策の効果検証・レポート作成が毎回大変
数字を集めて整理して資料にまとめる作業に、分析の時間が食われている
```

- レイアウト・カード形式・絵文字スタイルは既存の項目に合わせる
- 追加は1〜2個に留め、全体のバランスを崩さないこと

---

### 改善⑤：「6areas Categories」の表記を修正

**場所：** ヒーローセクションの数字3つが並ぶ部分（96% / 15+ / 6areas）

**内容：**
英語の文法が不自然なため、以下のいずれかに修正する。

- 修正案A：`6 Categories`
- 修正案B：`6領域`（日本語統一）

どちらが既存デザインのトーンに合うか判断のうえ採用すること。ラベルテキスト（"Categories" または "カテゴリー"）も合わせて調整する。

- ナビゲーション構造（Works / Skills / About / Contact）は変更しない
- 既存のカラーパレット・フォント・余白設計を踏襲する
- モバイル（スマートフォン）表示での確認を必ず行う
- 追加 CTA のリンク先は全て `https://forms.gle/avxAvwP63AWKmj4L9` で統一する
